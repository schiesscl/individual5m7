Código usado con cursor:
with connection.cursor() as cursor:
    cursor.execute("SELECT id, nombre, precio, disponible FROM productos_producto")
    
    # Obtener nombres de columnas
    columns = [col[0] for col in cursor.description]
    
    # Convertir filas en diccionarios
    for row in cursor.fetchall():
        productos_data.append(dict(zip(columns, row)))
¿Qué es un cursor?

Un cursor es un objeto que permite ejecutar consultas SQL y recuperar resultados fila por fila.

  • cursor.execute(sql) - Ejecuta la consulta
  • cursor.fetchone() - Obtiene una fila
  • cursor.fetchall() - Obtiene todas las filas
  • cursor.fetchmany(size) - Obtiene un número específico de filas
  • cursor.description - Información sobre las columnas
✅ Ventajas de usar cursores
  • Control total sobre la consulta SQL
  • Eficiente para grandes volúmenes de datos
  • Permite usar características específicas de MySQL
  • Útil para procedimientos almacenados
❌ Desventajas de usar cursores
  • No aprovecha el ORM de Django
  • Código menos portable entre bases de datos
  • Mayor probabilidad de errores
  • No obtiene instancias del modelo automáticamente
